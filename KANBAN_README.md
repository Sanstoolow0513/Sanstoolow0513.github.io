# 看板管理系统使用说明

## 简介

这是一个现代化的看板管理系统，采用黑白主题配色，提供类似Trello的核心看板功能与待办事项管理功能。系统支持数据持久化存储，所有数据保存在浏览器本地存储中。

## 功能特性

### 1. 看板管理
- **创建看板**：点击"新建看板"按钮，输入看板名称即可创建
- **编辑看板**：点击"管理看板"按钮，在看板列表中点击编辑图标
- **删除看板**：在管理看板界面点击删除图标（至少保留一个看板）
- **切换看板**：在管理看板界面点击看板名称即可切换

### 2. 列表管理
- **创建列表**：在看板右侧点击"添加列表"按钮
- **编辑列表**：鼠标悬停在列表上，点击编辑图标
- **删除列表**：鼠标悬停在列表上，点击删除图标

### 3. 待办事项管理
- **添加待办事项**：在列表底部点击"+ 添加待办事项"按钮
- **编辑待办事项**：点击待办事项卡片或编辑图标
- **删除待办事项**：点击待办事项卡片上的删除图标
- **查看详情**：点击待办事项卡片可查看完整详情

### 4. 待办事项详情
每个待办事项支持以下信息：
- **标题**：待办事项的主要描述
- **描述**：详细的任务说明
- **截止日期**：设置任务截止时间
- **创建时间**：自动记录创建时间

### 5. 拖拽功能
- 支持将待办事项在不同列表之间拖拽移动
- 拖拽时会有视觉反馈效果
- 支持在同一列表内重新排序

## 使用指南

### 快速开始

1. **访问系统**
   - 从主页点击"看板系统"按钮进入
   - 或直接访问 `kanban.html`

2. **创建第一个看板**
   - 点击页面顶部的"新建看板"按钮
   - 输入看板名称（如"项目开发"、"个人任务"等）
   - 点击"保存"完成创建

3. **添加列表**
   - 在看板右侧点击"添加列表"按钮
   - 输入列表名称（如"待办"、"进行中"、"已完成"等）
   - 点击"保存"完成创建

4. **添加待办事项**
   - 在列表底部点击"+ 添加待办事项"
   - 填写标题、描述和截止日期
   - 点击"保存"完成添加

### 高级功能

#### 拖拽移动待办事项
1. 鼠标按住待办事项卡片
2. 拖动到目标列表
3. 松开鼠标完成移动

#### 管理多个看板
1. 点击"管理看板"按钮
2. 在弹出的列表中查看所有看板
3. 点击看板名称切换当前看板
4. 使用编辑和删除按钮管理看板

#### 编辑现有内容
- **看板**：在管理看板界面点击编辑图标
- **列表**：鼠标悬停在列表上，点击编辑图标
- **待办事项**：点击卡片或编辑图标

## 界面说明

### 顶部导航栏
- 左侧显示系统标题
- 右侧显示"返回主页"按钮

### 看板选择区
- 显示当前看板名称
- "新建看板"按钮：创建新看板
- "管理看板"按钮：管理所有看板

### 看板区域
- 横向排列所有列表
- 每个列表包含标题、待办事项和添加按钮
- 最右侧是"添加列表"按钮

### 弹窗界面
- **看板弹窗**：创建或编辑看板
- **列表弹窗**：创建或编辑列表
- **待办事项弹窗**：创建或编辑待办事项
- **看板管理弹窗**：查看和管理所有看板

### 提示信息
- 操作成功或失败时会在右下角显示提示
- 成功提示显示为绿色
- 错误提示显示为红色

## 数据存储

- 所有数据保存在浏览器本地存储（localStorage）中
- 刷新页面后数据不会丢失
- 清除浏览器缓存会删除所有数据
- 建议定期备份重要数据

## 响应式设计

系统支持多种设备：
- **桌面端**：完整的横向布局
- **平板端**：自动调整布局适配屏幕
- **移动端**：垂直堆叠布局，优化触摸操作

## 快捷键

- **Enter键**：在输入框中按Enter可快速保存
- **拖拽**：鼠标拖拽移动待办事项

## 注意事项

1. 至少需要保留一个看板
2. 删除操作不可恢复，请谨慎操作
3. 截止日期过期时会显示红色标记
4. 当天截止的日期会显示橙色标记

## 常见问题

**Q: 数据会丢失吗？**
A: 数据保存在浏览器本地存储中，正常使用不会丢失。清除浏览器缓存会导致数据丢失。

**Q: 可以导出数据吗？**
A: 当前版本不支持导出功能，建议定期截图或手动备份重要数据。

**Q: 可以在手机上使用吗？**
A: 可以，系统完全支持移动端，提供优化的触摸体验。

**Q: 如何删除所有数据？**
A: 在浏览器设置中清除网站数据，或使用开发者工具清除localStorage。

**Q: 支持多人协作吗？**
A: 当前版本为单用户版本，不支持多人实时协作。
